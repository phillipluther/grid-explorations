8pt Hard Grid with 4pt Text Baseline

This is a super-simplified implementation of a grid system to demonstrate the basic concepts. Element spacing, margins/padding, and line heights snap to an 8pt grid; text baselines snap to a 4pt vertical grid.

It's a demo. It is not robust, nor complete, nor a plug-and-play system ready for anybody to pick up and start app'ing. The demo CSS includes only the stuff needed to demonstrate a few points and start discussions.

And Actually, While We're Making Caveats ...

You'll notice the text baselines can be 1px off above/below the text's rendered baseline. For simplicity, I'm using system fonts. Good ol' sans-serif and serif. I have no idea what those fonts might be or how they might render on your system.

Also as mentioned, I didn't fudge any of the numbers. Another way of saying it, I didn't finesse any of the numbers. Given time it's entirely possible to fine tune those 1px oddities and make everything snap exactly to the baseline grid.

Key CSS Callouts

This is an 8pt grid at a 1x pixel ratio; that's a high-falutin' way of saying it's an 8 pixel grid. Everything - margins, padding, widths, heights (including line heights) - occur as multiples of 8px.

Considerations

  • Font size doesn't affect the grid; line-height does
  • I don't want my grid to break if a user changes their base font size; that's not via CMD+/CTRL+, mind you, but if they actually set their font size to something different than the browser default of 16px. This is a web accessibility consideration.

Base/Default Font Size

The first thing I did was set my base font-size, as it'll serve as the basis for doing all other grid calculations. 

html { font-size: 1em; }

NOTE:

All browsers that matter use 16px as their default (unstyled) font size. I set the base font-size to 1em, effectively saying, "Use the browser default, which is probably 16px, to create the rest of our grid."

So yes - this could end up being a grid that's not based on 8px if a user sets their default font size to something bigger or smaller. This is an accessibility consideration.

Even if the default font size changes, though, everything will still grid properly - just at a larger or smaller scale. Eg., a user sets his/her default font size to 24px because s/he has trouble reading small print. That user will get a 12px grid.
